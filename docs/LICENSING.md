# Licensing

## Current status

No final license file has been added yet.

The intended posture is non-permissive/strong copyleft, but the exact license should be explicitly approved by the user before adding `LICENSE` to the repository.

## Recommended default

Recommended candidate: `AGPL-3.0-only`.

Reason:

- SE-CLI is intended to be open but not permissive.
- The system includes network/server behavior.
- Strong copyleft better matches the stated product posture than MIT, Apache-2.0, BSD, or ISC.

## Required approval before adding LICENSE

Adding or changing a repository license is an elevated mission.

A future mission should ask explicitly:

> Approve adding AGPL-3.0-only as the repository license?

Only after approval should the full license text be committed as `LICENSE` and referenced in `README.md`.

## Do not do

- Do not silently add a permissive license.
- Do not silently add a proprietary/no-license posture.
- Do not alter license terms as part of an unrelated code mission.
